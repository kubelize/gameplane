GamePlane Documentation

Learn how to create, configure, and manage game servers with GamePlane.

Getting Started

GamePlane makes it easy to deploy and manage dedicated game servers on Kubernetes. Follow these steps to get your first server running:

  1. Choose Your Game: Select from our supported game types
  2. Configure Resources: Set CPU, memory, and storage requirements
  3. Customize Settings: Configure game-specific options
  4. Deploy: Click "Create Server" and wait for deployment
  5. Connect: Use the provided connection details to join your server
Tip: Start with the default resource settings and scale up as needed based on player count and performance.

Supported Games

SDTD 7 Days to Die

Survival horror with tower defense elements.

  • Default ports: 26900-26902
  • Recommended: 2 CPU, 4GB RAM
  • Storage: 20GB minimum

Viking-themed survival and exploration game.

  • Default ports: 2456-2458
  • Recommended: 2 CPU, 4GB RAM
  • Storage: 10GB minimum

Creature collection and survival game.

  • Default port: 8211
  • Recommended: 4 CPU, 8GB RAM
  • Storage: 20GB minimum
Conan Conan Exiles

Open-world survival in the Conan universe.

  • Default ports: 7777, 27015
  • Recommended: 4 CPU, 8GB RAM
  • Storage: 50GB minimum

Configuration

Resource Requirements

Choose appropriate resources based on your expected player count and game requirements:

Player Count CPU Memory Storage
1-4 players 1-2 cores 2-4 GB 10-20 GB
5-10 players 2-4 cores 4-8 GB 20-50 GB
10+ players 4-8 cores 8-16 GB 50-100 GB

Network Configuration

  • LoadBalancer: Provides external IP access (recommended)
  • NodePort: Access via cluster node IP and specific port
  • ClusterIP: Internal access only

Troubleshooting

If your server is stuck in pending status, check:

  • Sufficient cluster resources (CPU, memory, storage)
  • Node affinity requirements (gaming-optimized nodes)
  • Storage class availability and binding mode
  • Image pull policies and registry access

Use kubectl describe pod to see detailed error messages.

Connection issues can be caused by:

  • Firewall blocking required ports
  • LoadBalancer not assigning external IP
  • Incorrect server password or settings
  • Server still starting up

Check the server logs for startup errors and verify the external IP is assigned.

To improve server performance:

  • Increase CPU and memory allocation
  • Ensure SSD storage is being used
  • Reduce player count or world size
  • Check for resource limits and requests

Monitor resource usage in the server details to identify bottlenecks.

API Reference

GameServer Custom Resource

Example GameServer manifest:

apiVersion: gameplane.kubelize.io/v1alpha1
kind: GameServer
metadata:
  name: my-game-server
  namespace: default
spec:
  gameType: sdtd
  serverName: "My 7DTD Server"
  serverDescription: "A fun survival server"
  resources:
    cpu: "2"
    memory: "4Gi"
    storageSize: "20Gi"
  networking:
    serviceType: LoadBalancer
  gameConfig:
    worldName: "Navezgane"
    difficulty: 1
    maxPlayers: 8

Status Fields

  • status.phase: Current deployment phase (Pending, Running, Failed)
  • status.externalIP: External IP address for connections
  • status.ports: Service port mappings
  • status.playersOnline: Current player count
